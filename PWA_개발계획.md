# 일정관리 PWA 개발 계획

## 기획안 개요

사무실에서 사용하던 탁상용 달력은 일정과 메모를 기록하는 데 편리하지만 사무실 밖에서는 접근이 어렵습니다.
사용자는 데스크톱과 휴대폰에서 모두 사용할 수 있는 **PWA(Progressive Web App)** 형태의 일정관리 앱을 원합니다.
이 앱은 인터넷 연결이 불안정하거나 없는 상황에서도 동작해야 하며, 책상 위 달력을 대체할 만큼 직관적인 월간 뷰를 제공해야 합니다.

---

## 연구 결과 및 중요한 기능

### PWA 기본 요소

- **서비스 워커(Service Worker)** – PWA의 핵심 기능으로 오프라인 지원, 캐싱, 백그라운드 동기화 및 푸시 알림을 담당한다. 서비스 워커는 앱과 네트워크 사이에서 동작하는 프로그래머블 프록시이다.
- **웹 앱 매니페스트(Web App Manifest)** – 앱의 이름, 아이콘, 표시 방식(standalone) 등을 정의하는 JSON 파일로 브라우저가 웹 사이트를 설치 가능한 앱으로 인식하도록 돕는다.
- **HTTPS 보안 연결** – 서비스 워커를 등록하려면 HTTPS 환경이 필수이며, 이는 사용자의 데이터와 신뢰성을 보장한다.
- **오프라인 강건성** – PWA는 네트워크 상태에 관계없이 동작하도록 설계해야 한다. 오프라인 시에는 캐시된 리소스와 로컬 데이터베이스를 사용하고 온라인 상태가 되면 서버와 동기화한다.

### 오픈 소스 시간관리 도구의 핵심 기능

- **모듈형 설계** – 일정 관리, 노트 작성, 알림 등 여러 기능을 모듈화하여 필요에 따라 켜거나 끌 수 있다.
- **통합 인증** – 여러 도구를 연동할 때 단일 계정으로 로그인할 수 있도록 지원한다.
- **크로스 플랫폼** – 데스크톱, 웹, 모바일 앱에서 모두 일정을 관리할 수 있도록 설계해야 한다.
- **오프라인 지원 및 동기화** – 모바일 앱은 오프라인 상태에서도 일정을 추가·수정할 수 있어야 하며, 인터넷 연결이 복구되면 데스크톱과 자동으로 동기화된다.

### 오프라인 기능과 서비스 워커의 중요성

- PWA는 전통적인 웹사이트와 네이티브 앱 사이의 경계를 허물며, 네트워크 상태에 관계없이 빠르고 상호작용적인 경험을 제공한다.
- 서비스 워커는 네트워크 요청을 가로채 캐시를 관리한다. static 자산뿐 아니라 API 응답까지 캐싱 전략을 선택할 수 있다.
- 데이터 저장을 위해 **IndexedDB** 와 같은 클라이언트 측 데이터베이스를 사용할 수 있으며, CRUD 작업을 통해 구조화된 객체를 로컬에 저장한다.

## 제안 기능 및 UI/UX 설계

- **월간/주간/일간 뷰** – 탁상 달력과 유사한 월간 그리드 뷰를 기본으로 제공하고, 필요에 따라 주간 및 일간 뷰로 전환한다.
- **일정 추가/수정/삭제** – 날짜를 클릭하거나 '새 일정 추가' 버튼을 눌러 제목, 날짜, 시간, 설명 등을 입력하고 저장할 수 있다.
- **색상 및 카테고리** – 업무, 개인, 회의 등 카테고리별로 색상 코드를 설정해 한눈에 구분할 수 있다.
- **검색과 필터링** – 검색창을 통해 특정 키워드나 기간으로 일정을 필터링한다.
- **알림 및 푸시** – 중요 일정에 대해 푸시 알림을 설정하고 모바일에서 알림을 받을 수 있도록 한다.
- **오프라인 입력 및 동기화** – 인터넷이 연결되지 않은 상태에서도 일정을 추가·수정할 수 있고, 온라인이 되면 서버와 자동 동기화한다.
- **다국어 및 시간대 지원** – 사용자의 시간대(서울 UTC+9)를 기반으로 일정을 표시하고, 언어는 기본적으로 한국어로 제공한다.

## 시스템 설계 및 기술 스택 제안

### 프론트엔드

| 구성 요소 | 설명 |
|---|---|
| 프레임워크/라이브러리 | React, Vue.js 또는 순수 JavaScript. 데모에서는 FullCalendar 라이브러리와 서비스 워커를 사용 |
| UI 라이브러리 | Tailwind CSS 또는 Material Design 기반 컴포넌트 |
| 데이터 저장 | IndexedDB (구조화된 이벤트 데이터), LocalStorage (간단한 상태 유지) |
| 오프라인 기능 | Service Worker를 통해 핵심 자산과 API 응답을 캐싱. Background Sync API 활용 가능 |

### 백엔드 (선택 사항)

1. **Firebase Firestore 또는 Supabase** – 실시간 데이터베이스와 인증을 제공하여 개발을 단순화한다.
2. **자체 서버 + REST API** – Node.js(Express), Spring Boot 등을 사용해 일정 정보를 저장하고 JWT 기반 인증을 구현한다.
3. **CalDAV 서버/Nextcloud** – 표준 캘린더 프로토콜을 사용해 여러 클라이언트와 호환된다.

### 보안 및 배포

- 모든 통신은 HTTPS로 이루어져야 하며, 서비스 워커 등록을 위해 필수이다.
- 간단한 배포는 **Netlify, Vercel, GitHub Pages, Railway** 를 사용할 수 있고, 자체 서버를 구축할 경우 Nginx/Apache를 이용한다.

| 단계 | 담당 | 설명 |
|---|---|---|
| 요구사항 수집 | 기획 관리자 | 사용자가 원하는 기능(월간 뷰, 알림, 동기화 등)을 정리하고 우선순위를 설정 |
| 기술 스택 결정 | 개발자, 기획 관리자 | 프레임워크, 데이터베이스, 서버 여부 등을 결정 |
| UI/UX 프로토타입 설계 | 기획 관리자 | 탁상달력과 유사한 인터페이스를 스케치하고 사용자 테스트 진행 |
| 기능 개발 | 개발자 | 캘린더 렌더링, 일정 CRUD, 오프라인 저장, 서비스 워커 등록, 알림 기능 구현 |
| 데이터 동기화 구현 | 개발자 | 서버 또는 클라우드 데이터베이스와의 동기화 로직 구현 및 충돌 처리 전략 수립 |
| 테스트 및 검수 | 개발자, 기획 관리자 | 데스크톱/모바일 환경에서 PWA 설치, 오프라인 기능, UI 호환성, 알림 등 테스트 |
| 배포 및 사용자 교육 | 기획 관리자 | 사내 사용자들에게 설치 방법과 사용법을 안내하고 피드백을 수집해 개선 |

## PWA 데모 코드 및 사용 방법

### 프로젝트 구조

```
desk-calendar-pwa/
├── index.html       # 메인 HTML – 달력과 모달 UI 정의
├── style.css        # 기본 스타일 정의
├── app.js           # 이벤트 CRUD, 로컬 저장 로직
├── manifest.json    # PWA 메타데이터 (이름, 아이콘, 색상 등)
├── sw.js            # 서비스 워커 – 핵심 자산 캐싱 및 오프라인 지원
├── icon-192.png     # 앱 아이콘 (192x192)
└── icon-512.png     # 앱 아이콘 (512x512)
```

### 실행 방법

1. 로컬에서 테스트하려면 Python으로 간단한 웹 서버를 실행:
   ```bash
   cd desk-calendar-pwa
   python -m http.server 8000
   ```
2. 브라우저에서 `http://localhost:8000` 을 열면 일정 관리 앱이 표시된다.
3. 새 일정을 추가하려면 상단의 **"새 일정 추가"** 버튼을 누르거나 달력 날짜를 클릭한다.
4. 오프라인 상태에서도 기존 일정을 확인하고 새로운 일정을 추가할 수 있다.

## 결론

PWA는 설치형 앱과 웹 사이트의 장점을 결합하여 **사무실 밖에서도 접근 가능한 일정 관리 도구**를 제공한다.
서비스 워커와 IndexedDB를 이용해 오프라인에서도 동작하며, 웹 앱 매니페스트와 HTTPS를 통해 데스크톱과 모바일 모두에서 설치 가능한 형태로 제공된다.
